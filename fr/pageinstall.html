Installation

La librairie est écrite en C++20, et requiert une version de GCC >= 10.2.0.

Installation pour Ubuntu 20.04

Prérequis obligatoires

sudo apt-get install gcc g++ make pkg-config libfmt-dev libeigen3-dev
Avertissement
Si vous avez déjà GCC installé, il faut vérifier qu'il est au moins en version 10.2.0 (gcc –version), sinon il faut le mettre à jour.

Prérequis optionnels

Si vous souhaitez pouvoir générer des graphiques avec libtsd, il faut par ailleurs installer libpng et freetype :

sudo apt-get install libpng-dev libfreetype6-dev

Compilation et installation de libtsd (Makefile)

Une fois les dépendances installées (y compris libpng-dev et libfreetype6-dev), il suffit d'exécuter les commandes suivantes :

git clone tsdconseil/libtsd
cd libtsd/core
# Compilation des sources (par exemple, 8 threads pour aller plus vite)
make -j 8
# Installation de la librairie et des fichiers d'en-têtes vers les dossiers standards (<code>/usr/local/lib, /usr/local/include</code>) :
sudo make install

La librairie est alors disponible dans le dossier /usr/local/lib/libtsd.a et les en-têtes dans /usr/local/include/tsd.

Note
Au moment de la compilation (ligne make -j 8 ci-dessus), vous pouvez ajouter les définitions optionnelles suivantes :
  • LIBTSD_USE_PNG=0 : désactive la possibilité de générer des images PNG (et supprime donc la dépendance à libpng),
  • LIBTSD_USE_FREETYPE=0 : désactive la possibilité de générer du texte dans les images (et supprime donc la dépendance à freetype).
  • TESTS=1 : construction des tests de non régression (l'exécutable de test sera alors généré dans le dossier build/debug-linux/tests.exe, à appeler depuis la ligne de commande).

Utilisation de la librairie

Un exemple simple est proposé dans le dossier libtsd/ex-simple. Le Makefile est constitué ainsi :

LDFLAGS = -ltsd -lfreetype -lpng -lz -lfmt
CXXFLAGS = -g -march=native -fPIC -std=c++2a `pkg-config --cflags eigen3`
all:
g++ exlibtsd.cc $(CXXFLAGS) $(LDFLAGS)

Compilation et installation de libtsd (cmake)

Avertissement
La compilation avec CMAKE en est encore en cours de mise au point pour l'instant – les instructions ci-dessous sont incomplètes et ne fonctionnent pas.
Note
Au moment de la configuration, le logiciel cmake devrait détecter automatiquement si libpng, libfreetype, et libgtkmm-3.0-dev sont disponibles, et, le cas échéant, activer la possibilté de générer des graphiques.
git clone tsdconseil/libtsd
mkdir build
cd build
cmake ../libtsd
make -j 8
sudo make install
Note
To enable the autotests, add the following option to the cmake command: -DBUILD_TESTS

Installation pour Windows 10 (avec MSYS 2)

Prérequis obligatoires

  • Télécharger et installer l'environnement MSYS2, disponible à l'adresse suivante : https://www.msys2.org/ (sélectionner la version 64 bits).
  • Dans le dossier c:/msys64, lancer le programme "mingw64.exe"
  • Exécuter :
    pacman -Syu
  • Fermer le terminal, et en ouvrir un nouveau (mingw64.exe)
  • Exécuter la commande suivante :
    pacman -S mingw-w64-x86_64-gcc mingw-w64-x86_64-gdb mingw-w64-x86_64-make mingw-w64-x86_64-pkg-config mingw-w64-x86_64-eigen3 mingw-w64-x86_64-fmt

Prérequis optionnels

Et si vous souhaitez pouvoir générer et visualiser des graphiques :

pacman -S mingw-w64-x86_64-freetype mingw-w64-x86_64-libpng

Et si de plus vous souhaitez pouvoir les visualiser en temps réel :

pacman -S mingw-w64-x86_64-gtkmm3
