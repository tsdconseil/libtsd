libtsd Documentation

Libtsd is a C++ library for signal processing, based on the Eigen library, which enable to do vectorized computations in a simple and intuitive way. The libtsd library bring routines for signal processing (readable and intuitive code), the kind that can be usually found in Matlab, Scilab or Python environments, but with the advantages of C++, especially in terms of possible real-time processing.

The library consists in the following modules, requiring no other dependancies than Eigen (in version 3):

  • dsp : general function for signal proessing (abstract structures for filters, sources and data sinks),
  • dsp::filter : design, analysis and implementation of linear filters (LTI).
  • dsp::fourier : frequency domain processings.
  • dsp::tf : Time / frequency processings (STFT, CQT, wavelets, ...).
  • dsp::telecom : digital and analog modulations and demodulations.
  • dsp::stats : processings based on signals statistics (linear prediction, sub-spaces methods, ...)
  • dsp::time : functions and structures useful for processing dates and times, calendars, etc.

The following optional modules car also be used:

  • dsp::view : Simple plotting functions (dependancies : libpng and freetype).

Some more applicative modules are also being developped:

  • dsp::kalman : Kalman filtering (lineair and non linear),
  • dsp::apps::doa : Computings related to antenna arrays.

Code style

The guiding rule that has been choosen is to simplify the best possible the API, so as they are easily readable, and so as to minimize the risk of mistake or confusion for the user. This imply the following decisions:

  • The number of exposed C++ class is minimized, either by exposing when possible only simple function (C style), or using pointers to abstracts classes, so as to mask the implementation details.
  • Even if it limits the flexibility, templates are not systematically used, and when a single type (for instance 32 bits floating point) is possible, it is used directly. This enables to well distinguis the API (header files) from the implementation (source files).

Naming conventions

  • The functions, variables and methods are written in lower case, and the words are separated with an underscore (_) : a_function,
  • The classes and structures are written in camelcase: AClass,
  • The constants are written in upper case, the words being separated with underscores: A_CONSTANT.

Abstract classes

The abstract classes are instanciated by a method of the kind module_name_new(), which returns a smart pointer to a class instance (std::shared_ptr<T>, abbreviated in dsp::sptr<T>).
