libtsd Documentation

Libtsd is a C++ library for signal processing, based on the Eigen library, laquelle permet d'effectuer des calculs vectorisés de manière simple et intuitive. La librairie Libtsd vous apporte les facilités du traitement du signal à haut niveau (code lisible et intuitif), du type que l'on trouve habituellement avec Matlab ou Python, mais avec les avantages du C++, notamment en terme de fonctionnement possible pour des traitement en temps réels.

Elle est constituée des modules de base suivants, ne nécessitant aucune dépendance autre que Eigen (en version 3) :

  • dsp : fonctions générales pour le traitement du signal (définition des structures abstraites pour les filtres / sources / puits de traitement)
  • dsp::filter : design, conception et implémentation des filtres linéaires (LTI)
  • dsp::fourier : traitements dans le domaine fréquentiel.
  • dsp::tf : traitements temps / fréquences (STFT, CQT, ondelettes, ...).
  • dsp::telecom : modulations / démodulations numériques et analogiques.

Les modules optionnels suivant peuvent par ailleurs être utilisés :

  • dsp::view : Tracés de courbes et la visualisation des données (dépendances : libpng et freetype, pour l'affichage : libgtkmm 3).
  • dsp::audio : Lecture et écriture de fichiers audio (dépendance : libsndfile).

Style de code

La règle directrice qui a été choisie est de simplifier au maximum les API, afin de faciliter qu'elles soient le plus lisibles et claires possibles, et de manière à miniser le risque d'erreur ou de confusion pour l'utilisateur. Cela implique notamment les choix suivants :

  • Le nombre de classes C++ exposées est minimisé, soit en exposant dans la mesure du possible de simples fonctions, soit en exposant des classes abstraites, de manière à masquer les détails d'implémentations.
  • Quitte à limiter la flexibilité, les templates ne sont pas systématiquement utilisés, et quand un type (flottant 32 bits par exemple) paraît adéquat pour une large gamme d'utilisation, il est utilisé directement. Cela permet notamment de bien séparer l'API (fichiers d'en-tête) de l'implémentation.
